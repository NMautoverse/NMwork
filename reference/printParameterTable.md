# Print parameter table to console, flextable, latex, or (latex) pdf file

Print parameter table to console, flextable, latex, or (latex) pdf file

## Usage

``` r
printParameterTable(
  pars,
  engine = "kable",
  format,
  footnotes = NULL,
  script = NULL,
  file.mod,
  include,
  include.pattern,
  drop,
  drop.pattern,
  include.fix,
  caption,
  ci = "cov",
  rse.cov = TRUE,
  label.pmtables
)
```

## Arguments

- pars:

  A parameter table as generated by \`NMwork::createParameterTable()\`.

- engine:

  "kable", "pmtables" or "flextable".

- format:

  A format like "tex" or "pdf" for a preview or a file name for saving a
  file. What formats are available depends on \`engine\`.

- footnotes:

  Additional footnotes to include. Footnotes about theta
  transformations, abbreviations, and source paths will automatically be
  included.

- script:

  Path to source code to include in source code.

- file.mod:

  Path to Nonmem control stream. Normally when \`pars\` was created with
  \`createParameterTable()\`, this is not needed because \`pars\` will
  contain this information already. If so, file.mod should only be used
  in cases where modifications to the original parameter table are being
  performed to generate new ones.

- include:

  A character vector of symbol values to include. All else will be
  dropped.

- include.pattern:

  A regular expression run on the \`symbol\` column. What matches will
  be kept.

- drop:

  A character vector of symbol values to drop. All else will be kept.

- drop.pattern:

  A regular expression run on the \`symbol\` column. What matches will
  be dropped.

- include.fix:

  Either a logical or "NotZero". \`TRUE\` means fixed parameters are not
  filtered out, \`FALSE\` means they are filtered out, "NotZero" means
  they are only kept if not equal to zero.

- caption:

  If supplied, this text will be added to a simple model naming string,
  based on the \`file.mod\` file name.

- ci:

  The default ("cov") is to attempt to extract parameter estimation
  uncertainty from a succesful covariate step. To use a bootstrap
  estimate instead,

- rse.cov:

  Include relative standard error estimate based on covariate step?
  Default is TRUE if source=="cov" and FALSE if not.

- label.pmtables:

  If \`engine=="pmtables"\`, a latex label is needed. Like
  label.pmtables="tab:myfirsttable".

## Details

kable: for rmd and rmd-latex: can't get longtable to work with
footnotes. Currently not using footnotes.

pmtables is only available if the package called pmtables is installed.
pmtables is not available on CRAN, at the time of writing this. It can
be installed from MPN.

                                     format.sigma="%symbol - %label "))
printParameterTable(pars=pars,format="R")
